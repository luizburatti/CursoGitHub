
Comandos Git e GitHub

dir = lista todos os diretorios no windows.
Ls = lista todos os diretorios no linux.

CD = change director  comando para navegar nas pastas (linux e windows)  CD/ 
CD ..  = volta para diretorio anterior.

CLS - limpar terminal. 
Clear - limpar terminal (linux). Ctrl + L atalho para limpar o console.

Tab  =  Auto complete  (windows).

mkdir NomeDaPasta = Criar diretorio (mesmo comando para windows e linux).

echo - Printa o que foi escrevido. 
EX:
 Echo hello  = hello. 
> - direcionador de fluxo.

 Caso utilizar o  (echo hello > hello.txt) -- cria arquivo de texto pelo terminal.

 del - comando para deletar repositorio.(deleta tudo que tem dentro do repositorio).

 rmdir workspace(nome do diretorio)  /s /q  =  Deleta completamente o repositorio.
 rm -rf  workspace(nome do diretorio) = Deleta completamente o repositorio.(linux).


 sudo su -  solicita permição para instalar app no linux.
git --version - Busca a versão da app escolhida.


Fundamentos Git GitHub

SHA1 -  Encriptçao de 40 digitos.(segurança do git gub).

Echo "string" | openssl sha1 -- Incriptar mensagem.


Blobs/Trees/commits  -  3 Tipos basicos responsavel por versionamento.

echo 'conteudo' | git hash-object --stdin -- retorna sha1  utilizando o hash 

fc31e91b26cf85a55e072476de7f263c89260eb1

blob salva objetos  < no git 

echo -e 'blob 9\0conteudo' | openssl sha1

fc31e91b26cf85a55e072476de7f263c89260eb1

Tree -  armazena os blobs 

Commit > treee > Blobs   -- Junta toda estrutura dos meta dados.


Primeiros comandos GIT. 

Git init - iniciar o git dentro da pasta, para poder inicializar e 
poder usar o gerenciamento de codigo pelo git.

ls -a = mostra arquivos ocultos muito util para ver o arquivo do gitconfig.

Configuração do repositorio.

git config --global  user.email"email@hotmail.com"
git config --global  user.name Luiz  

git add *  - puxa todos os arquivos do repossitorio.

Git commit -m "comentario" -- Sobe nosso repositorio para o git.

Git status - tras o status do nosso repositorio.


$ mv Receita.html ./receitas/   -- Mover arquivos entre pastas.
  
git status - pastas untracked  pela movimentação de arquivos entre pastas.

Então irremos corrigir o repossitorio, adicionando novamente nosso arquivo de baseando 
no novo repositorio receitas/ 
git add Receita.html  receitas/

Git ADD - joga os arquivos para  Staged. 
Git commit - pega todos os arquivos  da staged e gera um batch para serem enviados.

Untracked, Unmodified, Modified e Staged.  4 Estagios de arquivo no git.

Trabalhando com repositorio local.

git config --list   -- Configuração  do seu git para o seu git hub email e etc.

git config --global --unset user.email  Apaga as config.

git config --global --unset user.name   Apaga as config.

repositorio local para repositorio servidor (github).

git remote add origin (repositorio) 
git remote add origin https://github.com/luizburatti/CursoGitHub.git

git remote -v    -- lista todos meus repositorios 

git push origin master   Envia para o repositorio os dados.

git pull origin master - puxar os dados do git hub 

git clone  link do repositorio.   -- trago o repositorio baixado do git hub de qualquer repositorio online.

 git submodule add PostGreSQL/digital_innovation_one  -- Salvar dados de outros repositorios no seu git hub
